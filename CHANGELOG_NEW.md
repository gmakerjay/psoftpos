## 👥 1. ระบบจัดการสมาชิกและแต้มสะสมใหม่ (Simplified Member & Points System)
* **การปรับปรุงฐานข้อมูล**:
  * เพิ่มคอลัมน์ `address` (ที่อยู่) และ `privilege` (สิทธิพิเศษ) ในตาราง `members`
  * เพิ่มคอลัมน์ `points_earned` (แต้มได้) และ `points_used` (แต้มใช้) ในตาราง `sales` สำหรับทำบันทึกประวัติการใช้แต้ม
* **การปรับปรุงหน้าจอหลักสมาชิก**:
  * แสดงข้อมูลที่จำเป็น 6 คอลัมน์หลัก: **ID, ชื่อสมาชิก, เบอร์โทร, ที่อยู่, สิทธิพิเศษ, และแต้มสะสม**
  * ยุบรวมและลบคอลัมน์ที่ซับซ้อนทิ้ง เช่น ระดับสมาชิก (Tiers), วันหมดอายุสิทธิ์, และยอดเงินสะสม
* **ฟอร์มเพิ่ม/แก้ไขสมาชิก**:
  * หน้าฟอร์มแบบใหม่รองรับการกรอกข้อมูล 5 ช่องหลักอย่างง่าย: ชื่อ-นามสกุล, เบอร์โทรศัพท์, ที่อยู่, สิทธิพิเศษ, และระบุแต้มสะสมเริ่มต้นได้ทันที
* **ระบบสะสม/หักแต้มในหน้าชำระเงิน POS**:
  * เพิ่มช่องกรอกและระบบจำกัดไม่ให้ใช้แต้มสะสมเกินจำนวนที่มีในหน้าต่างชำระเงินของ POS
  * เมื่อชำระเงินสำเร็จ ระบบจะตัดยอดและคำนวณแต้มสะสมใหม่ (เพิ่ม 1 แต้มต่อทุกๆ 100 บาทที่ซื้อ) พร้อมหักแต้มสะสมเดิม
  * ตารางแสดงประวัติซื้อสินค้าของสมาชิกจะโชว์คอลัมน์แต้มได้/ใช้ (`+แต้ม / -แต้ม`) เพื่อตรวจสอบย้อนหลังได้ร้อยเปอร์เซ็นต์
* **การทดสอบความถูกต้อง**:
  * เขียนสคริปต์ตรวจสอบการทำงานอัตโนมัติ `scratch/verify_member_system.py` ตรวจเช็คกระบวนการ CRUD และการตัดยอดแต้มใน Transaction ผลลัพธ์ **ผ่านฉลุย 19/19 การทดสอบ**

## 🌿 2. กรอบธรรมชาติสไตล์พรีเมียม (Premium App Window Natural Border)
* **สิ่งที่ปรับปรุง**:
  * ฝังรูปภาพขอบธรรมชาติสีเข้มลายใบไม้และลายไม้ `assets/app_border.png` เป็นพื้นหลังหลักในหน้าต่างแรกเริ่ม

## ⚡ 3. เพิ่มประสิทธิภาพสำหรับคอมพิวเตอร์สเปกต่ำ (Low-End PC Performance Optimizations)
* **สิ่งที่ปรับปรุง**:
  * **Garbage Collection**: รันตัวล้างข้อมูลหน่วยความจำขยะในพื้นหลังทุกๆ 5 นาทีด้วยระบบ `gc.collect()`
  * **Cache Clearing**: เคลียร์รายการสินค้าและรายชื่อสมาชิกใน RAM ทันทีที่ผู้ใช้กดสลับจาก POS ไปใช้งานเมนูอื่น ป้องกันแรมบวมสะสม
  * **Memory Leak Fix**: ล้างตัวแปรอ้างอิงเฟรม UI `_pos_frame` ทุกครั้งที่มีการเปลี่ยนแท็บเพื่อให้ตัว Garbage Collector สามารถทำลายและคืนแรมกลับระบบได้สมบูรณ์
  * **TclError Fix**: เพิ่มระบบตรวจสอบสถานะหน้าต่าง Tkinter ก่อนจะสั่งอัปเดตเวลานาฬิกาหลัก เพื่อป้องกัน Error แครชของ Console ตอนสั่งปิดโปรแกรม

## 🔑 4. แก้ไขข้อผิดพลาดระบบ License วันสุดท้ายใช้งานไม่ได้ (License Expiration Fix)
* **ปัญหาที่พบ**: ระบบตรวจสอบสิทธิ์เดิมเปรียบเทียบวันด้วยวินาทีทำให้เศษชั่วโมงวันสุดท้ายคิดเป็น `0` วัน และบังคับล็อกโปรแกรมไม่ให้เข้าใช้งานล่วงหน้า 1 วัน
* **สิ่งที่แก้ไข**: ปรับเปลี่ยนการคำนวณวันให้คิดระดับวันปฏิทิน (`date`) และยอมให้เปิดใช้งานโปรแกรมในวันสุดท้ายได้สำเร็จ (จะบล็อกสิทธิ์เมื่อวันคงเหลือ `< 0` วัน)
* **การทดสอบความถูกต้อง**: รันสคริปต์ `verify_license_system.py` และดักจับ attribute ใน parent module เพื่อความแม่นยำผลลัพธ์ **ผ่านฉลุย 39/39 การทดสอบ**

## 💻 5. แก้ไขความเข้ากันได้ wmic กับ Windows 11 (Windows 11 WMIC Compatibility Fix)
* **ปัญหาที่พบ**: เมื่อใช้ฟีเจอร์ล้างความจำเครื่องหรือตรวจสอบ Process ของ KeyGen เกิดข้อผิดพลาด exit status 1 และใช้งานไม่ได้ เนื่องจาก Windows 11 และ Windows 10 อัปเดตใหม่เอาเครื่องมือ `wmic.exe` ออก
* **สิ่งที่แก้ไข**: อัปเกรดโค้ดใน `keygen_standalone.py` และ `license_generator.py` ให้เช็ค Process ในระบบปฏิบัติการด้วย **PowerShell Native Query** (`Get-CimInstance Win32_Process`) ซึ่งทันสมัยและรองรับทุกเครื่องโดยอัตโนมัติ และใช้ระบบ `wmic` ดั้งเดิมเป็นทางเลือกสำรอง (Fallback) หากระบบไม่มี PowerShell
